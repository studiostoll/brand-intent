Design tokens capture decisions. Brand guidelines capture rules. Neither captures intent — the reasoning, the voice, the compositional logic that make a brand impossible to confuse with anything else. Brand Intent is an open file format that makes all of it machine-readable.

01 — The Problem

Slop is the default. Intent is the exception.

AI-generated content has a gravity toward the generic. Without specific constraints, every model reaches for the same superlatives, the same structures, the same performed enthusiasm that belongs to no brand in particular. What survives is specificity — brands that know exactly what they are, what they never say, and why every visual decision was made. That knowledge has to be authored.

Brand Intent is the missing layer: machine-readable authorship. Not rules — intent. The reasoning that makes a brand specific, in a format any AI agent can read and respect.

02 — The Framework

Every brand system answers five independent questions.

Most tools collapse them into one place. Brand Intent keeps them separate. Each question has its own file type. Each file type has one job. That separation is what makes the system composable.

02

03

05

The arrow is a derivation arrow. Identity contains reasoning. Brand derives expression intent from that reasoning. Format, Purpose, and Composition condition specific acts of expression without repeating what upstream layers establish.

03 — The Layers

Each file. One job. Nothing more.

Identity

The identity file is the only place brand strategy lives in machine-readable form. Voice, register, positioning, content pillars, audience definitions. Nothing visual. Nothing compositional.

The always / never construct is its most powerful primitive: concrete, parseable rules an AI can follow without interpretation. "Always say Sauerteig, never artisan bread" is more useful to a model than "brand voice: warm and authentic."

krume.identity
# Krume — Brand Identity
# Neighborhood sourdough bakery, Berlin-Neukölln.

essence:  Brot ist Handwerk, nicht Lifestyle. Mehl, Wasser, Zeit.
promise:  Wer reinkommt, riecht sofort, dass hier seit vier Uhr
          morgens jemand steht.

voice
  register:  informal, Du
  persona:   Bäckerin hinterm Tresen — direkt, trocken, stolz auf das Handwerk

  always:
    - Sauerteig, not "artisan bread"
    - konkrete Mehlsorten (Tipo 1, Roggen 1150) over "beste Zutaten"
    - Backzeit in Stunden, nicht "lange Teigführung"

  never:
    - Lifestyle-Sprache (kuratiert, handverlesen, Genussmoment)
    - "Artisan" oder "Craft" — wir sagen Handwerk
    - Englische Begriffe wenn ein deutsches Wort existiert

pillars
  primary:   Sauerteig, Handwerk, Zutaten, Kiez
  avoid:     Lifestyle-Gastronomie, Wellness-Ernährung, Influencer-Kultur

audience regulars
  label:       Stammkunden aus dem Kiez
  motivation:  ihr Brot, ihr Laden, verlässlicher Rhythmus
  language:    knapp, direkt, kein Erklären — die wissen Bescheid
This file contains no:
  • hex values or color references
  • font names or type sizes
  • composition rules or canvas specs

Brand

The derived expression intent of the identity. Colors, typography, operationalized voice parameters, content defaults. Not expression itself — the authored conditions that determine what valid expression looks like.

A hex value in a theme is not a color on screen — it is an intent that a renderer realizes. The identity says "persona: baker behind the counter, dry humor." The brand derives sentence-max: 8 and headline-pattern: noun-first from that.

krume.brand (excerpt)
brand-colors
  primary:   #2C1810  # dark roast
  accent1:   #C8A47E  # crust
  accent2:   #8B4513  # rye
  white:     #FAF6F0  # flour

theme Laden
  background:     $white
  text-primary:   $primary
  cta:            $accent2

typography headline
  weight:       700
  size:         6      # cqh
  autofit-min:  4
  autofit-max:  10

voice-tokens
  register:         informal
  sentence-max:     8
  headline-pattern: noun-first

Format

Platform, canvas dimensions, safe zones (danger / crop / comfort), grid spec, and which purposes are editorially valid for this surface. No colors. No typography. No composition rules.

Purpose

Defines a content type through semantic slots (primary, secondary, meta, label, cta). Uses identity-filter to narrow the composed upstream and identity-extension to add only what is genuinely slot-specific.

Composition

Slot-to-position mappings. Two modes: grid (explicit row/column placement) and flow (panels, stacks, sticky zones — content-driven arrangement). Zero brand-specific information. Compositions are the most reusable layer.

04 — For AI Agents

Your brand as infrastructure.

Brand Intent ships with a skill file — compatible with Claude Code, Cursor, Gemini CLI, and other AI harnesses — that teaches any agent how to parse and compose Brand Intent layers.

Once installed, an agent understands the derivation chain: how identity-filter narrows the composed upstream, how to validate content against voice.never and pillars.avoid. Brand knowledge becomes infrastructure — not a prompt you write each time.

Complementary to Impeccable: where Impeccable teaches agents how to design well, Brand Intent teaches them what this brand's expression intent is.

Install skills (once, org-wide)
$npx brand-intent install-skills
Add a brand to a project
$npx brand-intent add krume/brand

Claude Code · Cursor · Gemini CLI

05 — Get Started

Three steps to a machine-readable brand.

1. Install the VS Code extension

Syntax highlighting, autocomplete, and validation for all five file types.

extinstall stoll.bif-language

2. Install agent skills

Teach your AI harness how to read Brand Intent layers.

$npx brand-intent install-skills

3. Author your brand

Start with .identity — the reasoning, the voice, the always/never rules. Then derive the expression intent in .brand. Add formats for the platforms your brand inhabits. Purposes and compositions follow.

The Krume bakery brand is included as a fully commented reference implementation.

$npx brand-intent init --from krume
